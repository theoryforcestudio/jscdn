now they do...  (31/12/03 05:26:19)
    Can anyone get these links working again?

    Sure can, Stiletto :)

    Nowadays Fast/Alltheweb indexes the boards at fravia.net instead of 2113.ch
    The easiest solution would be to just cut out the 2113.ch part
    (and not replace it with fravia.net, cuz indexing might change again ).

    seekers board: url.all:num url.all:phplab/mbs.php3/mb001
    php board: url.all:num url.all:phplab/mbs.php3 -url.all:mb001

    The indexing of Ebenezer's doesn't seem to have changed.

    ebenezers board: url.all:num url.all:ebmb.org/mbs/

    BUT

    I've found that often FAR MORE results pop up when adding a counter-intuitive datefilter like "updated before 1-1-1980".
    Your milage may vary, adding extra keywords sometimes screws up this trick.
    I would suggest to try both with and without this extra filter.

    Examples:
    seekers with date filter gives ~5,300 results
    seekers without date filter gives ~700 results

    AllTheWeb uses some sort of tracking on the links which are followed from it, I suppose in order to improve accuracy, keep statistics, etc.

    Take for example, this thread linking to some Google "hacks" website.
    AllTheWeb finds the thread, and in the description of the thread, prints on the page that the URL is
    http://www.fravia.net/phplab/mbs.php3/mb001?num=-...1047273171
    where an ellipsis (...) takes the place of all characters from the middle to the end of the text string whose elimination will reduce the displayed URL to a total of <64 chars.
    If you then hover your cursor on the the resulting link in IE, you see what appears to be the "proper" URL,
    http://www.fravia.net/phplab/mbs.php3/mb001?num=-1&thread=1047273171

    However, if you follow the link AllTheWeb provides, it redirects improperly. The actual result URL is:
    http://www.alltheweb.com/r?ck_sm=3f80b183&rpos=1&rpge=1&ref=200320086&r=http%3A%2F%2Fwww.fravia.net%2Fphplab%2Fmbs.php3%2Fmb001%3Fnum%3D-1%26amp%3Bthread%3D1047273171
    And, I suppose, the messageboard script sees this somehow, and so it becomes:
    http://www.fravia.net/phplab/mbs.php3/mb001?num=-1&thread=1047273171
    Which, in turn, makes the page display
    http://www.fravia.net/phplab/mbs.php3/mb001
    because the URL is malformed.
    Somewhere along the line, either the browser or the website is making something "&" when it should be "&". But what is at fault? Optionally, is there a way of editing the messageboard so that it WILL work? (Laurent?)
