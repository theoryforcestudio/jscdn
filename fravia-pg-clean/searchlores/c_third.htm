Inside the unassembled code of this target there is a silly memory comparison between edi (real code for your inputted string) and eax (the code that you have inputted yourself as registration string) which is performed again and again every time you starte the target and try to register.
Thus actually you do not even need to check the algos: if you register as a single letter (old +ORC's trick) then your input string (try it) corresponds to the following:

  • c == 0x1bd52 i.e.: 114002
  • d == 0x1bcdb i.e.: 113883
Even a zombie would see that following this pattern if you register with the username "f" you would have as numberstring 0x77 (i.e. dec 119) less than what you would have had if you had registered as "d" (i.e. 113764, duh).
